# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in TermWatch, please report it responsibly.

**Do NOT open a public issue.** Instead, use [GitHub Private Vulnerability Reporting](https://github.com/chrismdemian/termwatch/security/advisories/new) or email the maintainer directly.

### What to expect

- Acknowledgment within 48 hours
- A fix or mitigation plan within 14 days for confirmed vulnerabilities
- Credit in the release notes (unless you prefer to remain anonymous)

## Scope

The following are in scope:

- Code execution vulnerabilities in the main or renderer processes
- IPC message injection or privilege escalation
- Preload script security issues
- Sensitive data exposure (credentials, tokens, session data)
- Dependencies with known CVEs

The following are out of scope:

- Vulnerabilities in third-party streaming sites loaded in the video view
- Issues requiring physical access to the machine
- Social engineering
